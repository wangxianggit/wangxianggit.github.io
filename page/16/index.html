numpy函数unpackbits

numpy函数unpackbits

  • unpackbits函数把整数转化成对应的二进制数。

  • int2binary = {} #整数到其二进制表示的映射
binary = np.unpackbits(
    np.array([range(10)],dtype=np.uint8).T,axis=1)
for i in range(10):
    int2binary[i] = binary[i]
print(int2binary)

Python库talib的安装

  • ​ TaLib官网：http://ta-lib.org/

  • Python wrapper for TA-Lib：http://mrjbq7.github.io/ta-lib/doc_index.html

  • pip安装talib时常常会出现如下错误：

    • error: Microsoft Visual C++ 14.0 is required. Get it with “Microsoft Visual C++ Build Tools”: https://visualstudio.microsoft.com/downloads/

  • 如果安装visual C++14.0往往安装包很大，安装起来很麻烦。

  • 因此可以用手动安装的方法：

    • 手动下载对应的whl文件，下载地址：https://www.lfd.uci.edu/~gohlke/pythonlibs/
    • 进入下载目录使用pip install filename.whl 安装

  • 命令行界面出现成功安装的提示就可以使用。

Hdf数据读取错误：cannot set WRITEABLE flag to True

Hdf数据读取错误：cannot set WRITEABLE flag to True

  • python中使用hdf5读取文件时报错：cannot set WRITEABLE flag to True of this array
  • 原因是numpy版本与pytables的最新版本不兼容，若numpy版本时1.16则版本太高。
  • 把numpy版本降到1.15即可：
    • python -m pip install numpy==1.15
  • 另外不仅出现在hdf数据读取会出现这样错误，其他类似报错也只需要把numpy版本降低即可。

ssh服务配置

ssh服务配置

  • 备份配置文件：cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak

  • 我们通过看配置文件发现ssh服务的默认端口是22，很多时候我们的服务器被黑不是被人针对了，而是对方在扫一个网段中开启22端口的机器，而机器密码又是很简单。

  • 把默认端口改了，这样就不容易被人扫出来。

  • 修改默认端口：

    • ~~~
      #Port 22
      Port 9527

  • 限制ssh监听的ip

    • 这个适用于服务器有多个IP，这样我们就可以只监听内网IP，这样就只能用在同一个局域网的机器去连接，而我们只需要让在同一个服务器的另一台机器监听在外网，这样就可以实现在外网也访问服务器了，方法如下

    • #ListenAddress 0.0.0.0
#ListenAddress ::
ListenAddress [ip]

SSH服务详解

ssh服务详解

  • 传统上的网络服务程序如ftp、pop、telnet等存在不安全因素：在网络上用明文传送口令和数据，很容易受到其他人的攻击，即安全验证方式很容易受到“中间人(man-in-the-middle)”的修改。
  • SSH 为 Secure Shell 的缩写，由 IETF 的网络小组（Network Working Group）所制定；
  • SSH 为建立在应用层基础上的安全协议。
  • SSH 是目前较可靠，专为远程登录会话和其他网络服务提供安全性的协议。利用 SSH 协议可以有效防止远程管理过程中的信息泄露问题。
  • SSH是标准的网络协议，可用于大多数UNIX操作系统，能够实现字符界面的远程登录管理，它默认使用22号端口，采用密文的形式在网络中传输数据，相对于通过明文传输的Telnet，具有更高的安全性。

两种认证方式

  • SSH提供了口令和密钥两种用户验证方式，这两者都是通过密文传输数据的。
  • 口令用户验证方式传输的是用户的账户名和密码，这要求输入的密码具有足够的复杂度才能具有更高的安全性。
  • 而基于密钥的安全验证必须为用户自己创建一对密钥，并把共有的密钥放在需要访问的服务器上。
  • 当需要连接到SSH服务器上时，客户端软件就会向服务器发出请求，请求使用客户端的密钥进行安全验证。
  • 服务器收到请求之后，先在该用户的根目录下寻找共有密钥，然后把它和发送过来的公有密钥进行比较。
  • 如果两个密钥一致，服务器就用公有的密钥加密“质询”，并把它发送给客户端软件。
  • 客户端收到质询之后，就可以用本地的私人密钥解密再把它发送给服务器。这种方式是相当安全的。

SSH服务认证

  • 基于口令认证：

    • 基于口令的安全验证的方式就是大家现在一直在用的，只要知道服务器的SSH连接帐号和口令(当然也要知道对应服务器的 IP及开放的 SSH端口，默认为22 ),就可以通过 ssh客户端登录到这台远程主机。
    • ssh [ip]后根据提示输入账户和密码即可。
    • 此时，联机过程中所有传输的数据都是加密的。

  • 基于密钥认证：

    • 首先在客户端生成一对密钥：ssh-keygen -t dsa

    • ~~~
