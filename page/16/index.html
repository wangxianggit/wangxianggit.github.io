shell字符串

shell字符串​

  • 单引号和双引号字符串：

    • 单引号里的任何字符都会原样输出，单引号字符串中的变量是无效的；

    • 单引号字串中不能出现单独一个的单引号（对单引号使用转义符后也不行），但可成对出现，作为字符串拼接使用。

    • 双引号里可以有变量，双引号里可以出现转义字符

    • ~~~
      name=’runoob’
      str=”My name is "$name"! \n”
      echo -e $str

  • 字符串长度：

    • name="runoob"
echo ${#name} #计算变量长度

shell变量删除替换

shell变量定义规则

  • 命名只能使用英文字母，数字和下划线，首个字符不能以数字开头。
  • 中间不能有空格，可以使用下划线（_）。
  • 不能使用标点符号。
  • 不能使用bash里的关键字（可用help命令查看保留关键字）。​
  • 变量与变量内容以一个等号=来连结，等号两边不能直接接空格符
  • 变量内容若有空格符可使用双引号”或单引号’将变量内容结合起来，但双引号内的特殊字符如 $ 等，可以保有原本的特性，单引号内的特殊字符则仅为一般字符 (纯文本)
  • 可用跳脱字符\将特殊符号(如 [Enter], $, , 空格符, 等)变成一般字符
  • 若该变量为扩增变量内容时，则可用”$变量”或${变量} 累加内容：PATH="$PATH":/home/bin
  • 若该变量需要在其他子程序运行，则需要以 export 来使变量变成环境变量：export PATH
  • 通常大写字符为系统默认变量，自行配置变量可以使用小写字符
  • .取消变量的方法为使用 unset ：unset var
  • 变量的显示和重定向可用：echo
  • 只读变量用readonly命令声明：readonly var

变量的删除替换

变量配置方式 说明
${变量#关键词} 若变量内容从头开始的数据符合『关键词』，则将符合的最短数据删除
${变量##关键词} 若变量内容从头开始的数据符合『关键词』，则将符合的最长数据删除
${变量%关键词} 若变量内容从尾向前的数据符合『关键词』，则将符合的最短数据删除
${变量%%关键词} 若变量内容从尾向前的数据符合『关键词』，则将符合的最长数据删除
${变量/旧字符串/新字符串} 若变量内容符合『旧字符串』则『第一个旧字符串会被新字符串取代』
${变量//旧字符串/新字符串} 若变量内容符合『旧字符串』则『全部的旧字符串会被新字符串取代』

  • 变量删除示例：

    • path="/usr/bin:/usr/local/bin:/usr/local/sbin"
echo ${path#/*usr/bin:} //将第一个目录删除，*号是通配符
echo ${path#/*:} //同样删除第一个目录
echo ${path##/*:} //保留最后一个目录
echo ${path#/*usr/local/bin:}//同样保留最后一个目录
echo ${path%%:*sbin}//保留第一个目录

Linux命令dmesg

Linux命令dmesg​

  • 参考链接：https://www.runoob.com/linux/linux-comm-dmesg.html
  • linux内核将开机信息存储于ring buffer，开机后使用dmesg命令查看。

  • 语法：dmesg [-cn][-s <缓冲区大小>]

  • 参数说明：

    • -c 　显示信息后，清除ring buffer中的内容。
    • -s<缓冲区大小> 　预设置为8196，刚好等于ring buffer的大小。
    • -n 　设置记录信息的层级

  • 开机信息亦保存信息于文件/var/log/dmesg中

  • 实例：

    • dmesg | grep 'VLAN'
dmesg | grep -n 'VLAN'//显示行数

jupyter notebook服务器配置

基本使用方法

  • 参考链接：https://jupyter-notebook.readthedocs.io/en/stable/index.html
  • ​ jupyter服务安装：
    • python版本要求：python3.3以上或python2.7
    • 安装anaconda，jupyter notebook也同时安装
    • 或在安装完python后使用pip安装：pip install jupyter
  • 启动jupyter：命令行下使用命令jupyter notebook
  • 打开一个指定文件：jupyter notebook <filename>.ipynb
  • jupyter notebook帮助：jupyter notebook --help

服务器配置

  • 安装后默认启动ip：http://127.0.0.1:8888

  • 配置文件路径：

    • Windows: C:\Users\USERNAME\.jupyter\jupyter_notebook_config.py

    • OS X: /Users/USERNAME/.jupyter/jupyter_notebook_config.py

    • Linux: /home/USERNAME/.jupyter/jupyter_notebook_config.py

    • 若没有相应的配置文件目录，需要手动生成：

      • jupyter notebook --generate-config

  • 密码设置：

    • 在命令行下使用命令：jupyter notebook password

      • ~~~
        jupyter notebook password
        Enter password: **
        Verify password: **
        [NotebookPasswordApp] Wrote hashed password to /Users/you/.jupyter/jupyter_notebook_config.json

    • 在命令行下启动jupyter notebook即可密码登录，运行。
