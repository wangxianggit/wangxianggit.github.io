C++数组的引用和引用的数组

C++数组的引用和引用的数组

  • 数组变量array向右结合，相当于(int&)array[n]，在这里array是一个数组，其中数组的每个元素是引用，即引用的数组。
  • 数组的引用中首先和&结合，即(int &array)[n]，这里array是引用，其引用的对象是数组。
  • #include <iostream>

using namespace std;

void Test(int (&array)[5])
{
    cout << "size:" << sizeof(array) << endl;
    for (int i = 0; i < sizeof(array)/4; i++)
    {
      cout << array[i] << " "<<endl;
    }
}
int main()
{
    int  array[5] = {1,2,3,4,5};
    Test(array);
    int n3[3]={2,4,6};
    int (&rn3)[3]=n3;
    for(int i=0;i<3;i++)
      cout<<n3[i]<<"  "<<rn3[i]<<"  "<<endl;
    return 0;
}

C++关键字const_cast

const_cast用法

  • const_cast是一种C++运算符，主要是用来去除复合类型中const和volatile属性（不是真正去除）。

  • 使用方法：

    • const_cast<type>(expression)

C/C++中的引用

C/C++引用

  • 引用，即某一个变量或对象的别名，对引用的操作与对其所绑定的变量或对象的操作完全等价。

  • 语法格式：

    • ~~~
      变量类型 &引用名=目标变量名；

  • 引用相当于变量或对象的别名，因此不能再将已有的引用名作为其他变量或对象的名字或别名

  • 引用不是定义一个新的变量或对象，因此内存不会为引用开辟新的空间存储这个引用

    • ~~~
      #include
      using namespace std;
      int main(){

      int val=10;
int &n_val=value;
cout<<"val地址为："<<&val<<endl;
cout<<"n_val地址为："<<&n_val<<endl;
return 0;

      }

      output:
      val地址为：0x207d040
      n_val地址为：0x207d040

  • 对指针的引用

    • #include<iostream>
using namespace std;
int main(){
    int a=10;
    int *ptr=&a;
    int *&new_ptr=ptr;
    cout<<&ptr<<" "<<&new_ptr<<endl;
    return 0; 
}

编译器cc、gcc、g++和CC区别

编译器cc、gcc、g++和CC

  • gcc 是GNU Compiler Collection，原名为Gun C语言编译器，因为它原本只能处理C语言，但gcc很快地扩展，包含很多编译器（C、C++、Objective-C、Ada、Fortran、 Java），可以说gcc是GNU编译器集合。
  • g++ 是C++编译器。
  • cc 在 Unix系统下是 C Compiler，一个是古老的 C 编译器。在Linux 下 cc 一般是一个符号连接，指向 gcc；可以通过 $ ls -l /usr/bin/cc 来简单察看，该变量是 make 程序的内建变量，默认指向 gcc 。 cc 符号链接和变量存在的意义在于源码的移植性，可以方便的用 gcc 来编译老的用cc编译的Unix软件，甚至连 makefile 都不用改在，而且也便于 Linux 程序在 Unix下 编译。
  • CC 一般是 makefile 里面的一个名字标签，即宏定义，表示采用的是什么编译器（如：CC = gcc）。

注意事项

  • 后缀为.c的，gcc把它当作是C程序，而g++当作是c++程序；后缀为.cpp的，两者都会认为是C++程序，注意，虽然C++是C的超集，但是两者对语法的要求是有区别的。C++的语法规则更加严谨一些。
  • 编译阶段，g++会调用gcc，对于C++代码，两者是等价的，但是因为gcc命令不能自动和C++程序使用的库联接，所以通常用g++来完成链接，为了统一起见，干脆编译/链接统统用g++了，因此给人一种以为cpp程序只能用g++的错觉。
  • gcc不会定义__cplusplus宏，而g++会，实际上，这个宏只是标志着编译器将会把代码按C还是C++语法来解释，如上所述，如果后缀为.c，并且采用gcc编译器，则该宏就是未定义的，否则，就是已定义。
  • 编译可以用gcc/g++，而链接可以用g++或者gcc -lstdc++。因为gcc命令不能自动和C++程序使用的库联接，所以通常使用g++来完成联接。但在编译阶段，g++会自动调用gcc，二者等价。

C/C++关键字const

C/C++ 关键字const

  • const关键字用于修饰变量使之具有不能被修改的属性。

修饰局部变量

  • 修饰局部变量使用方法

    • ~~~
      const int a = 10;
      or
      int const b = 20;

  • 常量指针指向的值不能改变，但是指针本身可以改变，即常量指针可以指向其他的地址。

  • 指针常量是指指针本身是个常量，不能在指向其他的地址，即其指向的地址不能改变。

修饰引用

  • ~~~
    int a = 1;
    int const &a = b;
    const int &a = b;
  • 如果给以“指针传递”方式的函数返回值加 const 修饰，那么函数返回值（即指针）的内容不能被修改，该返回值只能被赋给加const 修饰的同类型指针。

const常量和宏常量的区别

  • const常量有数据类型，而宏常量没有数据类型。编译器可以对前者进行类型安全检查，而对后者只进行字符替换，没有类型安全检查，并且在字符替换时可能会产生意料不到的错误（边际效应）。
  • const定义常量从汇编的角度来看，只是给出了对应的内存地址，而不是象#define一样给出的是立即数，所以，const定义的常量在程序运行过程中只有一份拷贝，而#define定义的常量在内存中有若干个拷贝。
  • 宏定义是一个“编译时”概念，在预处理阶段展开，不能对宏定义进行调试，生命周期结束于编译时期。const常量是一个“运行时”概念，在程序运行时使用，类似于一个只读数据。
  • 编译器通常不为普通const常量分配存储空间，而是将它们保存在符号表中，这使得它成为一个编译期间的常量，没有了存储与读内存的操作，使得它的效率也很高
  • const修饰符可以防止意外的修改，增强程序的健壮性。
  • 宏定义不能作为参数传递给函数；const常量可以在函数的参数列表中出现。
