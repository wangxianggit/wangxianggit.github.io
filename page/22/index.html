归并排序算法

分治思想

  • ​ 许多有用的算法在结构上是递归的：为了解决一个给定的问题，算法一次或多次递归调用其自身以解决紧密相关的若干子问题。
  • 这些算法遵循分治法的思想：将原问题分解为几个规模较小的但类似于原问题的子问题，递归地求解这些子问题，然后再合并这些子问题的解来建立原问题的解。
  • 分治模式在每层递归时都有三个步骤：
    • 分解原问题为若干子问题，这些子问题是原问题的规模较小的实例。
    • 解决这些子问题，递归地求解各子问题，若子问题的规模足够小，直接求解。
    • 合并这些子问题的解即原问题的解。

归并排序

  • 归并排序算法遵循分治模式：

    • 分解：分解待排序的n个元素的序列成n/2个元素的两个子序列。
    • 解决：使用归并排序递归排序两个子序列。
    • 合并：合并两个已经排序的子序列已产生已排序的答案。

  • 归并排序算法的关键操作是合并步骤中的两个已排序的序列，通过调用一个辅助过程MERGE(A,p,q,r)来完成合并，伪代码如下：

    • ~~~
      MERGE(A,p,q,r)

      n1=q-p+1
      n2=r-q
      let L[1..n1+1] and R[1..n2+1] be new array
      for i=1 to n1
      L[i]=A[p+i-1]
      for j=1 to n2
      R[j]=A[q+j]
      L[n1+1]=null
      R[n2+1]=null
      i=1
      j=1
      for k=p to r
      if L[i]<=R[j] 

      A[k]=L[i]
i=i+1

      else A[k]=R[j]

      j=j+1

  • 为此对于任意长度的数组，当长度为2的幂次方时，可以运用自底向上的思想，依次合并长度为1的数组到长度为2的数组，合并长度为2 的数组到长度为4的数组，直到所有元素排序。

插入排序

  • ​ 伪代码：

    • INSERTION-SORT(A)

for j=2 to A.length
  key=A[j]
  //Insert A[j] into the sorted sequence A[1..j-1]
  i=j-1
  while i> and A[i]>key
    A[i+1]=A[i]
    i=i-1
  A[i+1]=key

shell命令df和du

df和du区别

  • Linux中du命令通过搜索文件并计算出每个文件的大小后累加计算容量，同时du能看到的文件是当前已经存在的，删除后的文件看不到。
  • df命令和du命令不同，通过文件系统获取占据空间的大小，同时如果删除一个文件后df也可以看到，因为在系统中当执行删除命令后文件并不会马上消失，而是暂时消失当所有应用不用时才根据系统的规则释放文件。
  • 若文件系统彻底删除后，du和df的效果是一样。

df命令

  • 指令df可以显示现有的文件系统的可用空间及使用情况，格式：df [option][file]

  • 例如：

    • df -h

Centos7 httpd基本操作

httpd基本操作

目录文件

  • 基本配置文件：
    • /etc/httpd/conf/httpd.conf
    • /etc/httpd/conf.d/*.conf
  • 模块配置文件：
    • /etc/httpd/conf.modules.d/
    • /usr/lib/systemd/system/httpd.service //service start
  • 程序启动：/usr/sbin/httpd
  • 日志文件：/var/log/httpd
    • access_log访问日志
    • error_log错误日志
  • 模块文件路径：/usr/lib64/httpd/modules

基本操作

  • 开启httpd服务：systemctl start httpd
  • 重启httpd服务：systemctl restart httpd
  • 设定开机启动：systemctl enable httpd
  • 查看状态：systemctl status httpd
  • 下载apache用户手册：yum install httpd-manual
    • 安装后手册位于目录：/usr/share/httpd/manual
    • 将手册复制到apache设定默认目录下可通过ip访问：http://主机ip/manual
  • 查看httpd进程：ps -aux | grep httpd
  • 结束httpd进程：killall -9 httpd 或./apachectl stop

shell特殊符号

shell特殊符号

  • ；分号，连续运行命令：echo "hello";echo "world"
  • | 管道，前面命令的标准输出作为后面命令的标准输入：ps -e | grep "ssh"
  • & 将命令放到后台执行
  • && 前面命令返回值为0才执行后面的命令:ls && echo "ok"
  • || 前面命令返回值为非0才执行后面的命令:lls || echo "ok"
  • # 号:
    • - 单个井号表示注释
- $# 表示位置参数的个数:echo $#
- ${#变量名} 表示变量的长度:a="hello";echo ${#a}
- ${#变量名[@]} 表示数组的个数:a=(1 2 3);echo ${#a[@]}
  • !惊叹号:
    • 将命令或者条件表达式的返回值取反
    • 执行历史命令: !231
    • vi或者ftp中执行外部shell命令
  • $符号：
    • 取变量值
    • 正则表达式或一些程序中表示行尾
  • >大于号，输出重定向：echo ‘abc’ > a.txt
  • <小于号，输入重定向：
  • 双圆括号，算术运算符：echo $((10/3))
  • []单方括号：
    • 通配符和正则中表示匹配括号中的任意一个字符
    • 条件测试表达式：[-f /etc/passwd]
    • 数组中下标括号: echo ${a[0]}
  • [[]]双方括号，字符串比较测试：[[a=b]]
