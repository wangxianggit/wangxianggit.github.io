cmake常见概念

cmake常见概念

target

  • target:cmake 的主要工作大多是为了操作各种各样的 target

  • cmake 把二进制可执行文件和库都称为 target。

  • target 就是指 cmake 要编译的目标

  • 可执行文件：add_executable()指定了可执行文件，它是 unix 上 a.out 之类的文件和 windows 平台 .exe 文件

库

  • 动态库和静态库都使用 add_library()命令生成，也就是 unix 平台上的 .so 和 .a 文件，window 上的 .DLL 文件等等。

  • add_library()默认生成 .a 文件，如果要生成 .so 文件用

  • add_library(test SHARED test.cpp)这样的形式，要指定 SHARED 模型，才会生成 libtest.so 文件。
    默认的效果等同于

  • add_library(test STATIC test.cpp)生成的是 libtest.a 文件。

  • link_libraries(hello test)cmake 通过 link_libraries() 命令指定了目标间的依赖关系，代码中 hello 是可执行文件，test 是库。

指定头文件

  • include_directories()指定了编译系统的头文件地址

cmake编译静态链接库

cmake编译静态链接库

  • 例如 hello.cpp 需要调用一个 so 中的 接口api，so 库是由 源文件world.cpp 编译生成。

  • 各个文件内容如下：

  • world.h 的代码，它位于工程中 ./include文件夹：

    • ~~~
      #ifndef WORLD_H__
      #define __WORLD_H

      void say_hello();

      #endif

  • hello.cpp的代码:

    • ~~~
      #include
      #include “include/world.h”
      using namespace std;
      int main(int argc,char** argv)
      {
      cout << “Hello for cmake!” << endl;
      say_hello();
      return 0;
      }

    • include_directories()这句代码指定了头文件地址

    • add_library()的作用是生成库文件，默认是 .a 的文件，也就是静态库

    • target_lingk_libraries()就是为可执行文件 hello 链接 libworld.a 这个库。

  • 执行 cmake .操作可以看到会正常生成 MakeFile 文件。

  • 执行make命令可以生成相应目标文件。

  • 目录下生成了两个文件 libworld.a 和 hello，即静态链接和目标文件。

  • cmake执行顺序为：指定头文件路径，指定动态库或者是静态库的路径，然后链接。

cmake生成简单makefile

cmake生成简单makefile

  • cmake是跨平台编译工具，自动化生成Makefile等编译文件。

  • 简单例子：

    • 现有源文件只包含一个文件hello.cpp:

    • ~~~
      #include
      using namespace std;

      int main(int argc,char** argv)
      {

      cout << "Hello for cmake!" << endl;
return 0;

      }

    • cmake_minimum_required()这句代码是指定 cmake 的最低版本
      project()这句代码是指定工程的名称
      add_executable()这个指示生成的可执行文件，上面的例子是编译 hello.cpp 生成 hello 这个可执行文件

    • CMakeLists.txt 写好后，在当前目录执行 cmake .命令，.代表在当前目录执行操作，如果 CMakeLists.txt 在别处，需要将路径添加在 cmake 后面。

    • 命令完成后，我们可以发现当前目录多了 1 个名为 CMakeFiles 的文件，也多了一个 Makefile 文件。

    • 执行 make 命令，最终可以发现生成了可执行文件 hello.

    • 在终端执行 ./hello，可以看见屏幕结果

keras框架参数epoch,batch_size

keras框架参数epoch,batch_size

  • epochs指的就是训练过程中数据将被“轮”多少次，即所有训练数据仅仅参与一次不够，需要多次参与训练达到loss下降的要求。
  • batch，梯度下降每次的参数更新有两种方式：
    • 第一种，遍历全部数据集算一次损失函数，然后算函数对各个参数的梯度，更新梯度。这种方法每更新一次参数都要把数据集里的所有样本都看一遍，计算量开销大，计算速度慢，不支持在线学习，这称为Batch gradient descent，批梯度下降。
    • 另一种，每看一个数据就算一下损失函数，然后求梯度更新参数，这个称为随机梯度下降，stochastic gradient descent。这个方法速度比较快，但是收敛性能不太好，可能在最优点附近晃来晃去，hit不到最优点。两次参数的更新也有可能互相抵消掉，造成目标函数震荡的比较剧烈。
  • 为了克服两种方法的缺点，现在一般采用的是一种折中手段，mini-batch gradient decent，小批的梯度下降，这种方法把数据分为若干个批，按批来更新参数，这样，一个批中的一组数据共同决定了本次梯度的方向，下降起来就不容易跑偏，减少了随机性。另一方面因为批的样本数与整个数据集相比小了很多，计算量也不是很大。
  • 基本上现在的梯度下降都是基于mini-batch的，所以Keras的模块中经常会出现batch_size，就是指计算一次梯度下降所用到的样本数。
