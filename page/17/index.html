jupyter notebook服务器配置

基本使用方法

  • 参考链接：https://jupyter-notebook.readthedocs.io/en/stable/index.html
  • ​ jupyter服务安装：
    • python版本要求：python3.3以上或python2.7
    • 安装anaconda，jupyter notebook也同时安装
    • 或在安装完python后使用pip安装：pip install jupyter
  • 启动jupyter：命令行下使用命令jupyter notebook
  • 打开一个指定文件：jupyter notebook <filename>.ipynb
  • jupyter notebook帮助：jupyter notebook --help

服务器配置

  • 安装后默认启动ip：http://127.0.0.1:8888

  • 配置文件路径：

    • Windows: C:\Users\USERNAME\.jupyter\jupyter_notebook_config.py

    • OS X: /Users/USERNAME/.jupyter/jupyter_notebook_config.py

    • Linux: /home/USERNAME/.jupyter/jupyter_notebook_config.py

    • 若没有相应的配置文件目录，需要手动生成：

      • jupyter notebook --generate-config

  • 密码设置：

    • 在命令行下使用命令：jupyter notebook password

      • ~~~
        jupyter notebook password
        Enter password: **
        Verify password: **
        [NotebookPasswordApp] Wrote hashed password to /Users/you/.jupyter/jupyter_notebook_config.json

    • 在命令行下启动jupyter notebook即可密码登录，运行。

Linux命令source

source 命令

  • source命令使当前的shell读入文件，并执行其中的shell命令，通常用于环境变量的修改。
    • 语法格式：source filepath 或 . filepath（点符号等同于source）
  • source命令不需要文件具有可执行权限，即对文件的权限没有限制。

和./、sh的区别

  • ./命令需要文件具有可执行权限，才可执行，./命令在进行脚本执行时会查找脚本第一行是否指定了解释器，如果没有指定即用当前系统默认的bash执行，否则将会交给指定的解释器。
  • sh命令也是执行某个脚本，和source命令一样不需要脚本具有可执行权限。
  • sh执行叫脚本时打开一个subshell去执行，并且在subshell里面设置的变量不会影响到父shell。

Linux find命令

Linux find命令

  • ​find命令可以搜索指定目录下的文件。

  • 语法：find path -option [expression]

  • 其中path表示要搜索的目录，option表示查找参数

  • 实例：

    • find //查找当前目录下的所有文件
find ./ -name "a.txt" //查找当前目录下名称为a.txt的文件
find ./ -maxdepth 2 -name "a.txt"//限制深度为2查找

Linux grep命令

Linux grep命令

  • 当前目录，搜索preamp字段的所有文件，并打印出所在行：

    • grep preamp *

output:
preamp.cpp:double preamp(double *x,double *par)

preamp.cpp~:double preamp(double *x,double *par)
preamp.cpp~:void preamp()
preamp.cpp~:  TF1 *fp=new TF1("fp",preamp,0,200,4);

详细使用方法

  • 语法：grep [-abcEFGhHilLnqrsvVwxy][-A<显示列数>][-B<显示列数>][-C<显示列数>][-d<进行动作>][-e<范本样式>][-f<范本文件>][--help][范本样式][文件或目录...]

  • 参数说明：

    • -a 或 --text : 不要忽略二进制的数据。
-A<显示行数> 或 --after-context=<显示行数> : 除了显示符合范本样式的那一列之外，并显示该行之后的内容。
-b 或 --byte-offset : 在显示符合样式的那一行之前，标示出该行第一个字符的编号。
-B<显示行数> 或 --before-context=<显示行数> : 除了显示符合样式的那一行之外，并显示该行之前的内容。
-c 或 --count : 计算符合样式的列数。
-C<显示行数> 或 --context=<显示行数>或-<显示行数> : 除了显示符合样式的那一行之外，并显示该行之前后的内容。
-d <动作> 或 --directories=<动作> : 当指定要查找的是目录而非文件时，必须使用这项参数，否则grep指令将回报信息并停止动作。
-e<范本样式> 或 --regexp=<范本样式> : 指定字符串做为查找文件内容的样式。
-E 或 --extended-regexp : 将样式为延伸的普通表示法来使用。
-f<规则文件> 或 --file=<规则文件> : 指定规则文件，其内容含有一个或多个规则样式，让grep查找符合规则条件的文件内容，格式为每行一个规则样式。
-F 或 --fixed-regexp : 将样式视为固定字符串的列表。
-G 或 --basic-regexp : 将样式视为普通的表示法来使用。
-h 或 --no-filename : 在显示符合样式的那一行之前，不标示该行所属的文件名称。
-H 或 --with-filename : 在显示符合样式的那一行之前，表示该行所属的文件名称。
-i 或 --ignore-case : 忽略字符大小写的差别。
-l 或 --file-with-matches : 列出文件内容符合指定的样式的文件名称。
-L 或 --files-without-match : 列出文件内容不符合指定的样式的文件名称。
-n 或 --line-number : 在显示符合样式的那一行之前，标示出该行的列数编号。
-o 或 --only-matching : 只显示匹配PATTERN 部分。
-q 或 --quiet或--silent : 不显示任何信息。
-r 或 --recursive : 此参数的效果和指定"-d recurse"参数相同。
-s 或 --no-messages : 不显示错误信息。
-v 或 --revert-match : 显示不包含匹配文本的所有行。
-V 或 --version : 显示版本信息。
-w 或 --word-regexp : 只显示全字符合的列。
-x --line-regexp : 只显示全列符合的列。
-y : 此参数的效果和指定"-i"参数相同。
