Markdown插入表格

  • ​ 默认情况下Markdown 插入的表格，单元格中的内容默认左对齐。

  • 普通表格markdown代码：

    • ```
      一个普通标题 一个普通标题 一个普通标题
      短文本 中等文本 稍微长一点的文本
      稍微长一点的文本 短文本 中等文本
    • 显示效果
左对齐标题 右对齐标题 居中对齐标题
短文本 中等文本 稍微长一点的文本
稍微长一点的文本 短文本 中等文本
  • 额外说明：
    • |、-、:之间的空格被忽略，内容和|间的空格被忽略。
    • 每行第一个|和最后一个|可省略，-的数量至少一个。
    • 对齐方式由以下代码控制： -:表示内容和标题栏居右对齐，:-表示内容和标题栏居左对齐，:-:表示内容和标题栏居中对齐。

Markdown grammar

Markdown grammar

标题

  • 类Setext：任何数量的等号和减号（+ -）
  • 类atx形式：1到6个#号表示对应1到6阶标题，也可以添加作为对称效果，后面需要一个空格

区块引用

  • 标记区块引用类似email中用>的引用方式。
  • 区块引用的嵌套：根据层次加上不同数量的>:

This is the first level of quoting

This is the nested blockquote

  • 区块引用内可以使用其他的Markdown语法，包括标题、列表、代码区块等
This is the first block

  1. This is the second block

  2. This is the third block

    return shell_exec(“echo $input | $markdown_script”)

列表

  • 无序列表使用星号、加号或减号作为列表标记。
  • 有序列表受用数字接着一个英文句点。

代码区块

  • 简单缩进4个空格或是一个制表符

  • Here is an example of AppleScript:

    tell application “Foo”

    ​ beep

    end tell

分隔线

  • 在一行中用三个以上的星号、减号、底线来建立一个分隔线，行内可以插入空格。

区段元素

  • 行内式链接在方括号后面紧接着圆括号并插入网址链接：

This is [an example](http://example.com/"title") inline link

  • 参考式的链接在链接文字的括号后面再加上另一个方括号，而在第二个方括号里面要填入用以辨识链接的标记：

This is [an example][id] reference-style link.

在文字的任意处把标记的连接内容定义出来：[id]:http://example.com/"title"

链接标签可以有字母、数字、空白和标点符号但是并不区分大小写

  • 隐式链接标记中链接标记等同于链接文字，只要在链接文字后面加上一个空的方括号：

[Google][] 定义连接内容：[Google]:http://google.com

I get 10 times more traffic from [Google]1 than from [Yahoo]2 or [MSN]3.

强调

  • Markdown使用星号和底线作为标记强调字词的符号，使用一个或两个符号包围的字词表示不用程度的强调。

代码

  • 标记小段行内代码用反引号包围起来 Use the printf() function
  • 代码区段内含有反引号，可以用多个反引号来开启和结束代码区段。
  • 以三个反引号开头得到一个代码区段。

图片

  • 行内式的图片语法为

    • ~~~
      ![Alt text](/path to/img.jpg)

  • 目前，Markdown无法指定图片的高度和宽度，如果需要可以使用普通的<img>标签

Python库requests使用方法

  • ​ requests第三方库安装：

    • ~~~
      pip install requests
      安装完成后导入模块：
      import requests

    • get还可以传递参数：

      • ~~~
        payload = {‘key1’: ‘value1’, ‘key2’: ‘value2’}
        r = requests.get(“http://httpbin.org/get", params=payload)

    • POST请求用来提交表单数据，即填写一堆输入框，然后提交，有参数的POST请求如下：

      • payload = {'key1': 'value1', 'key2': 'value2'}
r = requests.post("http://httpbin.org/post", data=payload)

    • put请求、delete请求、head请求、option请求等类似，可以参考官方网站：http://www.python-requests.org/en/master/

cmd新建文件

  • ​ 建立空文件方法
    • cd.>a.txt, cd. 相当于空操作重定向到文本文件，即建立空文件
    • copy nul a.txt, nul表示空设备，它不可见存在于每个目录中，可以看成是一个特殊的文件；一般可以把输出写入到nul，来达到屏蔽输出的目的。在此表示将空设备复制到文本文件中，即创建了一个空文件。
    • type nul>a.txt, 表示显示空设备的内容，并重定向到文本文件中。
    • echo a 2>a.txt, “2”表示错误输出的句柄，在此没有错误输出即创建空文件。
    • 即只要没输出并重定向到文件中就可以了。
  • 建立非空文件的方法
    • 同上面类似，只要将非空内容重定向到文件即可。
    • 常用命令是echo，或type或copy等

Python特殊变量__name__

  • Python脚本中常常有这样的语句：if __name__ == ‘__main__‘，用于执行某个脚本，但是这里涉及到一个python特殊变量 __name__ ，这里是在变量前后都加上了双下划线，所以是python定义的特殊变量。

  • 关于python需要解释以下几点：

    • Python的模块可以独立运行，同时模块之间也可以通过import互相引用
    • import的操作会执行所import的模块
    • __name__ 特殊变量就是标识模块的名字的一个系统变量，若当前模块是主模块，模块名字就是 __main__ ，如果模块是被导入， __name__的值为模块名字
    • 通过模块名称的判断可以确定当前模块是否是主函数，并执行相应操作

  • 示例代码

    • ~~~
      #Py1.py
      #!/usr/bin/env python
      def test():
      print ‘name = ‘,name
      if name == ‘main‘:
      test()

    • 执行结果：

    • __name__=__main__
Py1.py __name__=Py1

  • 通过结果可以知道，Py2.py直接执行，那么内建变量__name__的值为__main__，否则为模块的名字，通过这个特性可以在if语句里面添加测试代码，减少BUG，提高程序的可靠性。
