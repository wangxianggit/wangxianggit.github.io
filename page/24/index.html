Linux中git使用

关于git

  • git安装：

    • ~~~
      yum install git
      git –version

    • 配置文本编辑器

      • ~~~
        git config –global core.editor “Editor”

    • 将生成的id_rsa.pub文件中的内容添加到github上，文件可能不可见在终端在切换到.ssh目录下可见。

  • 提交到远程仓库：

    • git init //initialization
git add filename
git commit -m "message"
git remote add origin git@github.com:<用户名>/<仓库名>.git
git pull origin master //因为github建立仓库时会有readme.md文件，先要拷贝一份
git push -u origin master //将本地仓库链接到master分支上，你当然可以链接到其他分支
git push//上传你的本地仓库

    • 从远程仓库clone：git clone git@github.com:username/projectname.git

Linux使用百度网盘

Linux使用百度网盘

  • 使用谷歌浏览器的百度云插件，fork地址：https://github.com/acgotaku/BaiduExporter

  • 下载百度云资源推荐使用下载软件Aria2，软件安装管理员用户下执行：

    • ~~~
      yum install aria2

    • 两种下载模式：命令行和RPC serve模式。后者使用简单，开启后可以直接加载下载任务。

    • aria2软件开启：aria2c –conf-path=<PATH>,PATH是配置文件的绝对路径，最后一级是配置文件名称。

  • 同时开启百度网盘插件和aria2软件，在百度网盘网页版可以看到相应的RPC下载选项，单击后可以在aria2终端看到下载任务已经开启了。

英文SCI论文中时态一般现在时

一般现在时

  • 一般现在时表示现在的某种状况，或经常反复发生的行为和动作。
  • 时态中的状语常常为：always, usually, often,sometimes,every day(week,year…),once a week表示频率的词。

英文论文中的一般现在时

  • 在sci英文论文中常常用到的一般现在时的情况为:

    • 在摘要里，描述一个事实或者论文中的结论时：

      • ~~~
        This new transition is tentatively assigned to the decay of the 170Ir ground
        state. (prc.65.054310)
        Conclusions on nuclear structure are drawn from the experimental ata, aided by Geant4 simulations and a discussion on theoretical calculations. (prc.98.044307)

    • 结论中，阐述一个得到的确切结果,但如果有表示动作的词要用过去式如determined,observed等。

      • As a main result, an α-decay branch from the excited state at 17 keV in 219Ra is proposed to resolve a discrepancy between the evaluated level scheme and the experiment. (prc.98.044307)
Contemporary theoretical calculations on deformation, rotational states and the α decay justify the nuclear structure interpretation. (prc.98.044307)

    • 解释图表中的数据，符号和图像的时候以及说明列出的公式。

windows下安装hexo博客

windows下安装hexo博客

  • Hexo是一个静态博客的生成工具，可以在windows或linux平台上运行。
  • 在hexo博客中可以使用各种主题，同时主题的格式也可以自己定制。

需要安装的软件

  • 安装步骤：

    • 安装git，在网上下载相应版本的应用程序后安装，安装完git配置好用户名，邮箱以及github公钥并上传到自己的github网站上。

    • 安装node.js,在网站上下载好合适版本应用程序后安装，简单安装按照程序默认即可，安装好之后打开gitbash输入npm -v，如果出现相应版本号即说明安装成功同时也配置好了环境变量。

    • 安装markdown编辑器：由于hexo博客文本是用markdown格式生成的，合适的markdown编辑器有利于提高编辑效率，windowns下markdownpad(http://markdownpad.com/)，typora(https://www.typora.io/)等都是不错的选择。

    • 为了避免由于在windows下的换行符转义出现的问题需要，在gitbansh中禁用自动转换：

      • git init
rm -rf .git
git config --global core.autocrlf false//禁用自动换行

### 生成博客

  • 新建文件夹后，进入该文件夹后输入命令：hexo init,进行初始化

  • 初始化完成后会输入命令:hexo g生成静态博客

  • 输入命令hexo s可以在localhost:4000地址中看到示例博客

  • 定制自己的个性化主题需要进行远程部署，并可以在配置文件当中配置个性化主题。
