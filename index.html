epel源及epel-release安装

epel源及epel-release安装

  • RHEL及其衍生发行版如CentOS、SL为了稳定，官方的rpm repository提供的rpm包往往是很滞后的，因为这是服务器版本，安全稳定是重点。
  • 所以，官方的rpm repository提供的rpm包不够丰富，很多时候需要自己编译很麻烦，而EPEL恰恰可以解决这两方面的问题。
  • EPEL全称：Extra Packages for Enterprise Linux 。EPEL是由 Fedora 社区打造，为 RHEL 及衍生发行版提供高质量软件包的项目。装上了 EPEL之后，就相当于添加了一个第三方源。
  • 另外还有rpmfusion.org，rpmfusion主要为桌面发行版提供大量rpm包，而EPEL则为服务器版本提供大量的rpm包，而且大多数rpm包在官方 repository 中是找不到的。
  • 另外一个特点是epel绝大多数rpm包要比官方repository 的rpm包版本要来得新，如现在(2021.06)CentOS上安装的php，RHEL为了稳定性还在延用5.4版，而php 的最新版本已经到7.4.
  • epel源安装：yum install epel-release

MySQL忘记root密码后重置

  • 进入MySQL出现错误：ERROR 1045 (28000): Access denied for user ‘root’@’localhost’ (using passwor:yes)
  • 即密码出现错误，如果遗忘密码需要重置：
    • 首先设置跳过MySQL密码认证过程，vi打开/etc/my.cnf并在[mysqld]后面的任意一行添加’skip-grant-table’用来跳过密码验证。
    • 重启MySQL：systemctl restart mysqld;
    • 重启之后输入mysql即可进入mysql;
    • 回到终端直接输入mysql可以免密码直接登录MySQL终端；
    • 在MySQL终端输入use mysql切换数据库；
    • 然后更改root密码：update mysql.user set authentication_string=password(‘root’) where user=’root’ ;
    • 退出重新编辑my.cnf文件删掉skip-grant-tables 这一行，然后重启MySQL即实现root密码重置。
